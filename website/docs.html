使用文档

NewReader 是一款精致、安全的 RSS 阅读器，覆盖 macOS 与 iOS。本文档介绍从安装到高级配置的全部步骤。

安装

macOS

  1. 前往 GitHub Releases 下载最新 NewReader-macOS.zip
  2. 解压后将 NewReader.app 拖入 /Applications
  3. 首次运行时，系统提示「无法验证开发者」，请在 系统设置 → 隐私与安全性 中点击「仍要打开」

iOS

通过 Xcode 编译安装到设备。在 Xcode 中打开 Package.swift，选择 NewReaderiOS target，连接 iPhone 后运行。

添加订阅源

  1. 按 ⌘N 或点工具栏 + 按钮
  2. 输入 RSS / Atom / JSON Feed 地址
  3. 点击「订阅」，Feed 标题和文章将自动出现在侧栏

支持导入 OPML 文件：点侧栏 ⋯ → 导入 OPML，选择你的 .opml 文件。

订阅源管理

  • 刷新：右键 Feed → 刷新，或工具栏刷新按钮刷新全部
  • 重命名：右键 Feed → 重命名，在弹窗中输入新名称即可
  • 标为已读：右键 Feed → 标为已读，将当前 Feed 所有文章标记已读
  • 删除：右键 Feed → 删除，移除订阅源及其所有文章

配置 AI 摘要与翻译

NewReader 支持 OpenAI 兼容 API（适用于 OpenAI、DeepSeek、通义千问等）和 Anthropic Claude。

  1. 打开设置（⌘,）→ AI 标签
  2. 选择 AI 服务商
  3. 填写 API 端点、API Key、模型名称
  4. 可选：点击「测试连接」验证配置
  5. 点击「保存」
API Key 加密存储在系统 Keychain 中，明文不会写入磁盘。端点必须使用 HTTPS。

在文章中使用 AI

  • AI 摘要：阅读文章时点工具栏 ✨ 按钮，自动生成 3-5 句中文摘要
  • 批量摘要：侧栏 ⋯ → 批量 AI 摘要，为所有未读文章生成摘要
  • 翻译：工具栏 🌐 按钮展开语言菜单，选择目标语言即可翻译

语音朗读 (TTS)

双引擎支持，可随时切换：

引擎说明网络要求
Apple 系统语音macOS / iOS 内置引擎，支持中文普通话、英日韩离线可用
自定义 TTS API接入任意 TTS 服务商，音质自然流畅需要网络

在阅读文章时，点工具栏 🔊 按钮即可朗读（支持断点续播）。设置 → 语音 标签可切换引擎并配置自定义 TTS 的 Endpoint 和 API Key。

全文提取

部分 RSS Feed 只提供摘要而非正文。点工具栏 📄 按钮，NewReader 自动从原文链接拉取完整正文。

iCloud 跨设备同步

同一 Apple ID 设备间自动同步订阅源、文章和阅读状态。

前提条件

  • macOS 系统设置中登录 iCloud
  • 使用正式签名的 App（通过 GitHub Release 下载）

查看同步状态

设置 → 存储 标签中查看 iCloud 同步状态：

  • 🟢 已激活 — 同步正常，数据自动跨设备传输
  • ⚫ 未激活 — 未登录 iCloud 或签名缺失，使用本地存储

离线缓存

文章内容自动本地缓存，断网也能阅读。设置 → 存储 中可查看缓存大小并一键清除。

快捷键 (macOS)

快捷键功能
⌘N添加订阅
⌘,打开设置
⌘R刷新当前订阅源

安全设计

  • API Key 使用系统 Keychain 加密存储，不落盘明文
  • WKWebView 禁用 JavaScript + CSP 策略，阻止脚本注入
  • URL 仅允许 http/https 协议，拦截私有 IP / localhost 防 SSRF
  • HTML 消毒移除 script / iframe / 事件处理器 / javascript: 链接
  • OPML 导入文件大小上限 5MB，防内存耗尽
